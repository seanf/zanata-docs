4.6. Bugfix: Long strings slow down the operation of the translation editor

Cause:
Dynamic resizing of the translation cell was implemented as a loop.
Consequence:
Long strings in the translation cell caused the translation editor to become unresponsive as the resizing loop executed.
Fix:
The dynamic resizing code for the translation editor was rewritten to remove the loop.
Result:
Long strings do not cause the translation editor to become unresponsive.
More information:
For more information refer to the original bug report at https://bugzilla.redhat.com/show_bug.cgi?id=750956
